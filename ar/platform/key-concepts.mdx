فهم النموذج الأساسي يجعل اتباع كل دليل آخر أسهل. تشرح هذه الصفحة المفاهيم الأربعة الأساسية التي ستواجهها في جميع أنحاء المنصة.

## مساحات العمل

مساحة العمل هي الحاوية ذات المستوى الأعلى لمؤسستك. تحتفظ بجميع المشاريع والأعضاء والإعدادات ومعلومات الفوترة. ينتمي كل مستخدم إلى مساحة عمل واحدة على الأقل، وتعمل كل مساحة عمل بشكل مستقل مع تكوينها وبياناتها الخاصة.

لكل مساحة عمل **مالك (Owner)** واحد بالضبط، يمكنه نقل الملكية أو حذف مساحة العمل نهائياً من **Workspace Settings → Danger Zone**.

تُعرَّف مساحات العمل بمعرّف فريد (slug) يُستخدم في عناوين URL واستدعاءات API.

## المشاريع

المشاريع هي الوحدة الأساسية للعمل داخل مساحة العمل. لكل مشروع أعضاؤه وإعداداته وبياناته الخاصة. يمكنك إنشاء أي عدد تحتاجه من المشاريع، ويمكن لنفس المستخدم الانتماء إلى مشاريع متعددة بأدوار مختلفة في كل منها.

<Info>
  تتجاوز عمليات تجاوز الأدوار على مستوى المشروع الأدوار على مستوى مساحة العمل. يمكن منح مستخدم Viewer على مستوى مساحة العمل وصول Admin على مشروع معين، ويمكن ترقية Contractor خارجي إلى Consultant على مهمة واحدة.
</Info>

## الأعضاء والأدوار

يُعيَّن لكل عضو في مساحة العمل دور يتحكم فيما يمكنه رؤيته وفعله. تنقسم الأدوار إلى مجموعتين:

**أدوار مساحة العمل** — لفريقك الداخلي:

| الدور | الوصف |
|------|-------------|
| **Owner** | كل ما يمكن للمشرف فعله، بالإضافة إلى نقل الملكية وحذف مساحة العمل وتصدير بياناتها. واحد لكل مساحة عمل. |
| **Admin** | وصول كامل إلى الإعدادات والفوترة وجميع المشاريع. يمكنه دعوة الأعضاء وإزالتهم. |
| **Member** | يمكنه إنشاء وإدارة المشاريع التي ينتمي إليها. لا يمكنه الوصول إلى الفوترة أو الإعدادات على مستوى مساحة العمل. |
| **Viewer** | وصول للقراءة فقط إلى المشاريع التي يُضاف إليها. |

**الأدوار الخارجية** — للمتعاونين الميدانيين (Consultant، Contractor، Subcontractor، Supplier). ترى الأدوار الخارجية فقط المشاريع التي تم تعيينها لها صراحةً، ولكل منها مجموعة أذونات ذات نطاق محدود. يمكن للمقاولين من الباطن تعديل عناصر Issues وSite Diary وPunch List التي أنشأوها بأنفسهم فقط.

راجع [الأدوار والأذونات](/platform/administration/roles-permissions) للحصول على المصفوفة الكاملة والتجاوزات لكل مشروع وسجل الأدوار وتدفق نقل الملكية.

## بيانات اعتماد API

تسمح مفاتيح API للأدوات والخدمات الخارجية بالتفاعل مع مساحة العمل الخاصة بك برمجياً. يكون نطاق كل مفتاح محدوداً بمساحة عمل معينة ويمكن تقييده بالوصول للقراءة فقط أو للقراءة والكتابة.

تعامل مع مفاتيح API كأسرار—قم بتدويرها فوراً إذا تم كشفها. راجع مرجع API للحصول على تفاصيل المصادقة ووثائق نقاط النهاية.
